1. Prerequisito

Asegúrate de poner este proyecto en GitHub.

2. Introducción y primeros pasos

2.1. Crear fichero YAML de workflow

Se puede llamar, por ejemplo, github-actions-pycicd.yaml y los tienes que colocar dentro de la carpeta .github/workflows.

2.2. Contenido del fichero YAML

Escribe el siguiente contenido en el fichero .github/workflows/github-actions-pycicd.yaml. No te preocupes si no entiendes la sintaxis y los detalles del mismo. Leyéndolo te puedes hacer una idea, más o menos ligera, de lo que se va a hacer y cuándo. Más adelante entraré a explicarte cada uno de los atributos YAML de este fichero, propios de las GitHub Actions. 

name: GitHub Actions for pycicd
run-name: ${{ github.actor }} is testing out GitHub Actions 🚀
on: [push]
jobs:
  Explore-GitHub-Actions:
    runs-on: ubuntu-latest
    steps:
      - run: echo "🎉 The job was automatically triggered by a ${{ github.event_name }} event."
      - run: echo "🐧 This job is now running on a ${{ runner.os }} server hosted by GitHub!"
      - run: echo "🔎 The name of your branch is ${{ github.ref }} and your repository is ${{ github.repository }}."
      - name: Check out repository code
        uses: actions/checkout@v5
      - run: echo "💡 The ${{ github.repository }} repository has been cloned to the runner."
      - run: echo "🖥️ The workflow is now ready to test your code on the runner."
      - name: List files in the repository
        run: |
          ls ${{ github.workspace }}
      - run: echo "🍏 This job's status is ${{ job.status }}."

2.3. Ponlo en el repositorio

Asegúrate que el fichero YAML está en tu repositorio. Si no lo está, crea un commit y súbelo (push) a GitHub (repositorio remoto).

2.4. GitHub / Actions

Si vas a tu repositorio, en la pestaña Actions verás el workflow que acabamos de configurar.

Figure 1: Captura de pantalla de un workflow de GitHub Actions

En esta pestaña puedes ver los pasos que son procesados en cada triggered.

Cada vez que hagas un push verás que se lanza este workflow.

3. Plantillas de workflow de GitHub

Puedes usar y seleccionar plantillas para tus workflows. Cuando vayas a la pestaña Actions, al pulsar el botón de New workflow verás una serie de workflows sugeridos, como ves en esta captura:

Figure 2: Captura con sugerencias de workflow para proyecto en Python

A partir de ahí, puedes configurar un workflow haciendo clic en el botón Configure de la plantilla que se adapte a tus necesidades.

Para el ejemplo en el que estamos trabajando puedes usar y probar la plantilla llamada Python application. Observa el YAML que se genera y trata de comprender lo que va a hacer (algunas partes son intuitivas y, en otro caso, al menos, te acostumbrarás al formato YAML para especificar tareas a realizar).

Para aceptar la plantilla solo tendrás que comitear los cambios desde el botón Commit changes…
